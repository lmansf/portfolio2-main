Automating Fraud Detection

Fraud detection using rebalancing to handle class imbalance in transaction data. Final Prediction accuracy: 89.35%

While this project is closed now, if I were to explore fraud detection again, I'd want to do a realtime system. The major disadvantage of the current approach is that it's strictly reactive, meaning it's used to review transactions that have already been completed. It requires past examples of fraud to train a model, PII considerations, and doesn't prevent fraud directly. A realtime system uses machine learning and velocity checks to deny potential fraud before it happens.
