Owl Park E-Commerce Pipeline

A live mock webstore and agent workflows feed a real-time analytics pipeline that turns raw events into decision-ready BI models. The goal is simple: move from "we have data" to "we can act on it fast."

Ingest 4 scheduled n8n workflows plus live store actions.
Store Supabase captures raw operational events in real time.
Model Fabric Dataflow Gen2 builds STAR-oriented reporting tables.
Analyze Power BI dashboards and MCP-assisted natural-language analysis.

Stack: Supabase, Fabric (Dataflow Gen2, M, Warehouse, Data Lake), n8n, Copilot/Claude + Power BI MCP, Power BI.

Why This Pipeline Exists

  • Operational systems are fast for transactions, but hard to analyze at scale.
  • Warehouse models simplify joins and improve query speed for business teams.
  • Structured dimensional design makes BI more reliable, maintainable, and reusable.

Pipeline Flow

Phase 1: Ingest

Agents handle pricing, consumer purchases, restocking, and sellout logging. Store purchases directly influence agent behavior.

Phase 2: Store

Supabase receives transactions, weather impact, events, products, visit dates, and warehouse/sellout activity as the raw source layer.

Phase 3: Transform, Model, Store

Fabric pulls data on a real-time schedule, transforms it into dimensional/fact models, and writes to Warehouse (live) plus Lake (daily backup).

Phase 4: Analyze and Act

Dashboards and AI-assisted analysis drive schedule optimization and pricing-agent tuning inside fairness boundaries.

Current Status

Working Now

Agent cadence is tuned to maximize monthly token utilization without hitting limits, with priority on consumer and sellout workflows.

Next Metric

Dynamic pricing impact reporting is in progress.
