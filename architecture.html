Ecommerce Data Architecture

From AI-driven data generation to BI reporting in a live mock webstore pipeline.

Pipeline Summary

This project demonstrates why analytics pipelines matter: operational systems collect lots of data, but teams need clean, reliable, analytics-ready tables to make decisions quickly. The pipeline turns raw transactional activity into STAR-style reporting models that support business, marketing, accounting, and optimization workflows.

Tech stack: Supabase, Microsoft Fabric (Dataflow Gen2, M, Data Lake, Warehouse), n8n, Claude/Copilot + Power BI MCP, and Power BI.

Architecture Flow

Phase 1: Ingest

Four scheduled n8n workflows and a live mock webstore generate the core activity stream. Workflows cover dynamic pricing, consumer purchasing behavior, warehouse restocking, and sellout monitoring.

Phase 2: Store

Supabase serves as the raw event repository, receiving real-time writes from agents and store activity across transactions, products, weather impact, events, visit dates, warehouse activity, and sellout logs.

Phase 3: Transform, Model, Store

Microsoft Fabric pulls from Supabase on a real-time schedule using Dataflow Gen2 and shapes data into dimensional and fact layers organized by domain query groups.

Output storage is split by purpose: a real-time Data Warehouse for OLAP and a daily Data Lake backup for historical recovery and archiving.

Phase 4: Analyze

Modeled datasets power live Power BI dashboards and natural-language analysis via Claude + Power BI MCP for business goal tracking and n8n engine tuning.

Phase 5: Take Action

Insights are translated into operational changes: schedule optimization to maximize token usage per pay cycle and pricing-agent tuning that pushes revenue while staying inside critic-enforced fairness bounds.

Current Results

Dynamic pricing impact metrics are in progress.

Agent cadence is currently balanced to maximize monthly token utilization without exceeding limits, prioritizing the consumer and sellout workflows, then dynamic pricing, then lower-frequency tasks.
